# Workspace Agent Notes

This file is the lightweight workspace instruction layer for day-to-day implementation.

## Core Scope
- Focus on active product workflows, not historical cache-research workflows.
- Keep code, UI, and docs aligned in the same change set.
- Prefer clear, reversible changes over broad refactors.

## Canonical Documentation
- Internal source of truth: INSTRUCTIONS.md
- Lessons learned and mistakes: EXPERIENCES.md
- External handoff contract: EXTERNAL_EXPORT_READER_GUIDE.md
- Startup commands: PIPELINE.md

## Active Pages
- Editor: public/index.html
- Export Reader: public/export-reader.html
- Mesh Inspector: public/mesh-inspector.html
- Collision Test: public/collision-test-mode.html
- Full Viewer: public/full-viewer.html (paused placeholder)

## Runtime Policy
- Collision policy is sidecar-only for export-reader and collision-test runtime.
- Prefer mesh-collision-index.json plus per-mesh *.collision.json sidecars.
- Do not reintroduce render-geometry fallback collision into runtime flows.

## Export Policy
- Export routes are sidecar-collision-oriented.
- Keep RH transform contract files in export packages:
  - transform-conventions.json
  - entity-index-rh.json
  - entities/full.rh.json
- Keep visual-settings.json in export packages for reader visual parity.

## UI Consistency Policy
- Keep shared top-header navigation available across active pages.
- If a page is intentionally paused, keep it navigable and clearly labeled as paused.

## Startup Policy
- Primary local start command: npm run local
- Alternate command: start-localhost
- Default server URL: http://localhost:3015
- Runtime browser libs under public/lib are generated from node_modules and should stay untracked.

## Change Validation Checklist
- Validate edited files for syntax and diagnostics before finishing.
- Confirm documentation affected by behavior changes is updated immediately.
- Confirm no stale references to removed pages or removed legacy flows.

## Guardrails
- Avoid adding duplicate guide files with overlapping authority.
- Keep this file concise and operational; historical deep dives belong in EXPERIENCES.md.
- If a rule here conflicts with INSTRUCTIONS.md, INSTRUCTIONS.md wins.
